# 2026-07-11 - Sherlock TDA v2.0 Development

## Thread Dump Analyzer v2.0 - Major Update

### Key Changes Implemented
- **JFreeChart Integration**: 4 chart types added
  - Thread States Pie Chart (distribution across states)
  - Stack Depth Distribution Pie Chart (1-10, 11-20, 21-30, 30+ frames)
  - Thread Name Groups Bar Chart (grouped by type like HTTP, JVM, Application)
  - Top 10 Thread Pools Bar Chart (by pool distribution)
- **TID-based Cross-Dump Comparison**: Threads matched by ID (tid), not by name. All thread states tracked, not just RUNNABLE.
- **State Transitions**: Table showing threads that changed state between consecutive dumps, with TID, thread name, from/to state, dump index, and pool.
- **State History Line Chart**: Visualizes thread state counts across multiple dumps in a line chart.
- **Identical Stacks Detection**: Tab showing threads sharing exact same call stacks — indicates thread pool contention or saturation.
- **Long-Running Thread Detection (Corrected)**: Based on Master Prasad's correction — requires multiple dumps (5-10s apart). A thread is long-running only if it is RUNNABLE in ALL dumps AND points to the exact same top stack frame (class + method + line number) in every dump. Previous single-dump stack depth heuristic was incorrect.
