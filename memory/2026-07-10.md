## Long-Running Thread Detection - CORRECTED v1.2

Based on Master Prasad's screenshot from Google search results:
- Proper method: Take 3-4 thread dumps spaced 5-10 seconds apart
- Compare them
- A thread is long-running if:
  1. It remains in RUNNABLE state across ALL dumps
  2. It points to the EXACT SAME LINE of code (same class, method, line number) in every dump

Implementation updated in DumpAnalyzer.java:
- findLongRunningThreads(List<ThreadDump> dumps) - proper multi-dump detection
- haveSameTopFrame() - compares className, method, and line number
- Single-dump stack depth heuristic kept as fallback for backward compatibility

UI updated:
- Long Running tab now auto-detects using all loaded dumps
- Shows message when only 1 dump is loaded (falling back to heuristic)
- Multi-comparison dialog updated to show proper long-running threads
